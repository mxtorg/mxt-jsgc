### 3.2 模块职责说明

| DtoGenerator | 生成带JSR-303校验的DTO类 |

#### 3.2.2 util 模块
| 类名 | 职责 |
|------|------|
| NamingUtil | 路径转驼峰、类名/方法名生成 |
| TypeMapper | JSON Schema类型到Java类型映射 |

| GitUtil | Git仓库操作（克隆、提交、推送） |

## 4. 核心类设计

### 4.1 ApiGeneratorMojo (主入口)
```java
@Mojo(name = "generate-api-code", defaultPhase = LifecyclePhase.GENERATE_SOURCES)
public class ApiGeneratorMojo extends AbstractMojo {
    
    @Parameter(defaultValue = "${project.basedir}/src/main/resources/api-spec")
    private File specDir;
    
    @Parameter(defaultValue = "${project.basedir}/src/main/java")
    private File outputJavaDir;
    
    @Override
    public void execute() throws MojoExecutionException {
        // 1. 解析JSON Schema配置
        // 2. 调用各生成器生成代码
        // 3. 执行Git推送（如配置）
        // 4. 执行Maven发布（如配置）
    }
}
```

### 4.2 生成器接口设计
所有生成器实现统一的接口：
```java
public interface CodeGenerator {
    void generate(ApiConfig config, File outputDir) throws GeneratorException;
}
```

## 5. 数据流设计

### 5.1 代码生成流程
```
1. 读取JSON Schema文件
   ↓
2. OpenApiUtil解析为ApiConfig对象
   ↓
3. 按顺序调用各Generator:
   ├─ DtoGenerator (先生成DTO供其他引用)
   ├─ ControllerGenerator
   ├─ FeignClientGenerator
   ├─ MarkdownDocGenerator
   └─ ...其他生成器
   ↓
4. GitUtil提交推送代码
   ↓
5. ApiPackager打包发布
```

### 5.2 依赖关系
- ControllerGenerator → DtoGenerator (DTO类型引用)
- FeignClientGenerator → DtoGenerator (DTO类型引用)
- 所有Generator → OpenApiUtil (配置数据)

## 6. 关键技术方案

### 6.1 JavaPoet代码生成
使用JavaPoet的TypeSpec、MethodSpec、FieldSpec等API构建Java代码结构，优势：
- 类型安全
- 自动处理import
- 支持注解生成
- 代码格式化

### 6.2 JSON Schema解析扩展
OpenAPI Parser原生不支持git、gav等扩展字段，采用Jackson直接解析顶层扩展字段。

### 6.3 Git集成方案
使用JGit库实现：
- 克隆仓库（如需要）
- 添加生成的文件
- 创建提交
- 推送到远程仓库

### 6.4 Maven发布方案
使用Maven Invoker调用外部Maven进程：
- 动态生成pom.xml
- 执行mvn deploy
- 支持多仓库配置

## 8. 错误处理策略

### 8.1 错误分类
| 错误类型 | 处理方式 |
|---------|---------|
| 配置文件解析错误 | 抛出异常，停止执行，输出详细错误信息 |
| 代码生成错误 | 跳过该文件，记录错误日志，继续处理其他文件 |
| Git操作错误 | 记录警告日志，继续执行后续步骤 |
| Maven发布错误 | 记录警告日志，继续执行后续步骤 |

### 8.2 日志规范
- 使用Maven Plugin Log API
- 日志级别：DEBUG（详细流程）、INFO（主要步骤）、WARN（警告）、ERROR（错误）
- 日志格式：`[步骤] 描述`

## 9. 扩展性设计

### 9.1 插件化架构
通过SPI机制支持自定义Generator：
```java
public interface CodeGenerator {
    String getName();
    int getOrder();
    void generate(ApiConfig config, File outputDir);
}
```

### 9.2 配置扩展点
支持在JSON Schema中添加自定义扩展字段，通过配置项启用对应的Generator。
