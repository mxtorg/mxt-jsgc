## 2. Controller类生成规则

### 2.1 类注解规则
固定生成以下注解：
```java
@RestController
@Tag(name = "${schema.description}")
@Slf4j
public class ${SchemaName} {
}
```

### 2.2 方法命名规则
**规则**: httpMethod + 路径驼峰化

**路径驼峰化算法**:
1. 移除路径开头的'/'
2. 将'/'替换为空格
3. 转为驼峰命名（首字母小写）
4. 拼接HTTP方法（首字母小写）

**示例**:
| 路径 | HTTP方法 | 方法名 |
|------|---------|--------|
| /users/instance | post | postUserInstance |
| /users/instance | put | putUserInstance |
| /users/instance | get | getUserInstance |
| /api/v1/orders | get | getApiV1Orders |

### 2.3 映射注解规则
| HTTP方法 | 注解 |
|---------|------|
| post | @PostMapping("${path}") |
| put | @PutMapping("${path}") |
| get | @GetMapping("${path}") |
| delete | @DeleteMapping("${path}") |

### 2.4 入参注解规则

#### 2.4.1 POST/PUT请求体
```java
public ReturnType methodName(@RequestBody RequestType request)
```

#### 2.4.2 GET/DELETE请求参数
```java
public ReturnType methodName(@RequestParam("${paramName}") ParamType paramName)
```

### 2.5 Swagger注解规则
- summary非空：生成`@Operation(summary = "${summary}")`
- summary为空：不生成@Operation注解
- 类级别：`@Tag(name = "${schema.description}")`

### 2.6 方法体模板
固定生成以下内容：
```java
// todo 实现业务逻辑
log.info("${方法描述}，入参：{}", ${参数名});
return ${默认返回值};
```

**默认返回值规则**:
- Long: 1L
- Boolean: true
- String: ""
- Object: new XxxResponse()
- void: 无返回

### 2.7 Controller完整示例
```java
package org.demo.cloud.controller;

import io.swagger.v3.oas.annotations.Operation;
import io.swagger.v3.oas.annotations.tags.Tag;
import lombok.extern.slf4j.Slf4j;
import org.demo.cloud.dto.RegisterRequest;
import org.demo.cloud.dto.UserResponse;
import org.demo.cloud.dto.UserUpdateRequest;
import org.springframework.web.bind.annotation.*;

@Slf4j
@RestController
@Tag(name = "测试端点类")
public class DemoEndpoint {

    @PostMapping("/users/instance")
    @Operation(summary = "Register a new user")
    public Long postUserInstance(@RequestBody RegisterRequest request) {
        // todo 实现注册逻辑
        log.info("Register a new user，入参：{}", request);
        return 1L;
    }

    @PutMapping("/users/instance")
    @Operation(summary = "Upate a user")
    public Boolean putUserInstance(@RequestBody UserUpdateRequest request) {
        // todo 实现更新逻辑
        log.info("Upate a user，入参：{}", request);
        return true;
    }

    @GetMapping("/users/instance")
    @Operation(summary = "Get user by ID")
    public UserResponse getUserInstance(@RequestParam("id") Long id) {
        // todo 实现根据ID获取用户逻辑
        log.info("Get user by ID，入参：{}", id);
        return new UserResponse(id, "User fetched successfully");
    }
}
```

## 3. Feign客户端生成规则

### 3.1 类注解规则
```java
@FeignClient(
    name = "${info.title}",
    url = "${api.${info.title.lowerCamelCase}.url:}"
)
public interface ${info.title}Client {
}
```

### 3.2 方法签名规则
- 方法名：与Controller完全一致
- 参数列表：与Controller完全一致
- 返回类型：与Controller完全一致
- 注解：与Controller完全一致

### 3.3 FeignClient完整示例
```java
package org.demo.cloud.feign;

import org.demo.cloud.dto.RegisterRequest;
import org.demo.cloud.dto.UserResponse;
import org.demo.cloud.dto.UserUpdateRequest;
import org.springframework.cloud.openfeign.FeignClient;
import org.springframework.web.bind.annotation.*;

@FeignClient(
    name = "Demo",
    url = "${api.Demo.url:}"
)
public interface DemoClient {

    @PostMapping("/users/instance")
    Long postUserInstance(@RequestBody RegisterRequest requestBody);

    @PutMapping("/users/instance")
    Boolean putUserInstance(@RequestBody UserUpdateRequest requestBody);

    @GetMapping("/users/instance")
    UserResponse getUserInstance(@RequestParam("id") Long id);
}
```

## 4. DTO类生成规则

### 4.1 类注解规则
```java
@Data
@Schema(description = "${schema.description}")
public class ${SchemaName} {
}
```

**可选注解**:
- 无参构造：`@NoArgsConstructor`
- 全参构造：`@AllArgsConstructor`

### 4.2 字段生成规则
每个property生成一个私有字段：
```java
@${校验注解}
@Schema(description = "${字段描述}")
private ${JavaType} ${fieldName};
```

### 4.3 校验注解映射规则

| JSON Schema约束 | 条件 | JSR-303注解 |
|----------------|------|------------|
| required数组包含 | type=string | @NotBlank(message = "${fieldName}不能为空") |
| required数组包含 | type!=string | @NotNull(message = "${fieldName}不能为空") |
| minLength | type=string | @Size(min=${minLength}, message = "${fieldName}长度不能少于${minLength}位") |
| maxLength | type=string | @Size(max=${maxLength}, message = "${fieldName}长度不能超过${maxLength}位") |
| format=email | type=string | @Email(message = "${fieldName}格式不正确") |

### 4.4 类型映射规则
| JSON Schema类型 | format | Java类型 |
|----------------|--------|---------|
| string | - | String |
| string | email | String |
| string | date | LocalDate |
| string | date-time | LocalDateTime |
| integer | - | Integer |
| integer | int32 | Integer |
| integer | int64 | Long |
| number | - | Double |
| number | float | Float |
| number | double | Double |
| boolean | - | Boolean |
| array | - | List&lt;T&gt; |
| object | $ref | 引用的DTO类 |

### 4.5 DTO完整示例
```java
package org.demo.cloud.dto;

import io.swagger.v3.oas.annotations.media.Schema;
import jakarta.validation.constraints.Email;
import jakarta.validation.constraints.NotBlank;
import jakarta.validation.constraints.Size;
import lombok.Data;

@Data
@Schema(description = "注册请求体")
public class RegisterRequest {

    @NotBlank(message = "username不能为空")
    @Size(min = 3, message = "username长度不能少于3位")
    @Schema(description = "用户名")
    private String username;

    @NotBlank(message = "email不能为空")
    @Email(message = "email格式不正确")
    @Schema(description = "邮箱")
    private String email;

    @NotBlank(message = "password不能为空")
    @Size(min = 8, message = "password长度不能少于8位")
    @Schema(description = "密码")
    private String password;
}
```

## 5. 命名工具规则

### 5.1 路径转方法名
```java
/**
 * 路径转方法名
 * @param httpMethod HTTP方法（get/post/put/delete）
 * @param path 路径（如 /users/instance）
 * @return 方法名（如 getUserInstance）
 */
public static String pathToMethodName(String httpMethod, String path)
```

**算法步骤**:
1. 移除路径开头的'/'
2. 按'/'分割成多个片段
3. 第一个片段首字母小写，后续片段首字母大写
4. 拼接HTTP方法（首字母小写）

### 5.2 路径转类名
```java
/**
 * 引用路径转类名
 * @param ref 引用路径（如 #/components/schemas/DemoEndpoint）
 * @return 类名（如 DemoEndpoint）
 */
public static String refToClassName(String ref)
```

**算法步骤**:
1. 截取最后一个'/'之后的内容
2. 首字母大写（如果需要）

### 5.3 下划线转驼峰
```java
/**
 * 下划线转驼峰
 * @param str 下划线字符串（如 user_name）
 * @return 驼峰字符串（如 userName）
 */
public static String underscoreToCamelCase(String str)
```

## 6. 包路径规则

### 6.1 包结构
```
${gav.pkg}/
├── controller/       # Controller类
├── feign/           # Feign客户端接口
├── dto/             # Request/Response DTO类
└── config/          # 配置类（如需要）
```

### 6.2 包名生成
- Controller: `${gav.pkg}.controller`
- Feign: `${gav.pkg}.feign`
- DTO: `${gav.pkg}.dto`

## 7. 导入规则

### 7.1 自动导入
JavaPoet自动处理以下导入：
- 生成的类引用的其他DTO类
- 注解类
- 常用Java类型

### 7.2 固定导入
Controller固定导入：
- org.springframework.web.bind.annotation.*
- io.swagger.v3.oas.annotations.Operation
- io.swagger.v3.oas.annotations.tags.Tag
- lombok.extern.slf4j.Slf4j

FeignClient固定导入：
- org.springframework.cloud.openfeign.FeignClient
- org.springframework.web.bind.annotation.*

DTO固定导入：
- lombok.Data
- io.swagger.v3.oas.annotations.media.Schema
- jakarta.validation.constraints.* (如需要)

## 8. 代码生成顺序

为避免依赖问题，按以下顺序生成：
1. DTO类（被Controller和Feign引用）
2. Controller类
3. FeignClient类
4. 其他配置类
