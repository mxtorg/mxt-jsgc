## Lock文件机制检查

### 核心功能检查

- [ ] **Lock文件生成**: 代码生成完成后，在项目根目录生成`.api-generator.lock`文件
- [ ] **Lock文件格式**: Lock文件为有效JSON，包含`generatedAt`字段（ISO 8601格式）
- [ ] **Lock文件格式**: Lock文件包含`configMd5`字段（32位MD5哈希值）
- [ ] **Lock文件格式**: Lock文件包含`generatedFiles`字段（数组，包含所有生成的文件路径）
- [ ] **Lock文件格式**: Lock文件包含`generatorVersion`字段（插件版本号）

### MD5计算检查

- [ ] **MD5准确性**: 相同配置文件生成相同的MD5值
- [ ] **MD5差异检测**: 配置文件内容变化后，MD5值发生变化
- [ ] **MD5忽略空白**: 配置文件中的空白字符差异不影响MD5计算

### 增量生成检查

- [ ] **首次生成**: 项目目录不存在Lock文件时，执行完整代码生成
- [ ] **无变化跳过**: Lock文件存在且MD5一致时，跳过代码生成，输出提示信息
- [ ] **有变化重生成**: Lock文件存在但MD5不一致时，重新生成代码并更新Lock文件
- [ ] **强制重生成**: 配置`skipLockCheck=true`时，无论Lock文件状态如何都重新生成

### Lock文件管理检查

- [ ] **Lock文件忽略**: `.gitignore`文件包含`.api-generator.lock`条目
- [ ] **Lock文件读取失败处理**: Lock文件格式错误时，记录警告并重新生成
- [ ] **Lock文件不存在处理**: Lock文件不存在时，执行完整生成流程
